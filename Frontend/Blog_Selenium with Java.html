Selenium with Java: A Complete Tutorial

A step-by-step guide to using Selenium with Java for web automation testing.

Introduction to Selenium

Selenium is an open-source tool widely used in automation testing frameworks for web applications. It supports multiple programming languages such as Java, Python, C#, and JavaScript. Among these, Java is a popular choice due to its robustness and extensive libraries.

Selenium provides a suite of tools for automating web browsers and allows testers to validate web applications across different operating systems like Windows, Linux, and macOS.

Components of Selenium Suite

  • Selenium IDE: A browser extension for recording and running scripts.
  • Selenium RC: Interacts directly with browsers using programming languages.
  • Selenium WebDriver: The advanced version that controls browsers natively.
  • Selenium Grid: Enables parallel execution of tests across multiple machines.

Why Use Java with Selenium?

  • Strong community support with extensive documentation.
  • Robust libraries like TestNG and JUnit for test management.
  • Cross-platform compatibility — works seamlessly on different OS.
  • Integration flexibility with tools like Maven, Cucumber, and Jenkins.

Prerequisites

  • Eclipse IDE (with built-in JDK)
  • Maven
  • Browser drivers (e.g., ChromeDriver)

Setting Up Selenium with Java (Using Maven)

  1. Open Eclipse IDE.
  2. Create a new Maven Project (File → New → Maven Project).
  3. Add the Selenium dependency in your pom.xml file:
<dependencies>
<dependency>
<groupId>org.seleniumhq.selenium</groupId>
<artifactId>selenium-java</artifactId>
<version>4.8.0</version>
</dependency>
</dependencies>

Writing a Basic Selenium Test in Java

import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;

public class SeleniumTest {
  public static void main(String[] args) {
    System.setProperty("webdriver.chrome.driver", "path/to/chromedriver");
    WebDriver driver = new ChromeDriver();
    driver.get("https://www.google.com");
    System.out.println("Page Title: " + driver.getTitle());
    driver.quit();
  }
}

Locating Web Elements in Selenium

Selenium offers multiple locators to identify elements:

  • ID: driver.findElement(By.id("elementID"));
  • Name: driver.findElement(By.name("elementName"));
  • Class Name: driver.findElement(By.className("className"));
  • CSS Selector: driver.findElement(By.cssSelector("cssSelector"));
  • XPath: driver.findElement(By.xpath("XPathExpression"));

Handling Alerts and Pop-ups

import org.openqa.selenium.Alert;
import org.openqa.selenium.By;
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;

public class AlertHandling {
  public static void main(String[] args) {
    System.setProperty("webdriver.chrome.driver", "path/to/chromedriver");
    WebDriver driver = new ChromeDriver();
    driver.get("https://www.selenium.dev/documentation/webdriver/interactions/alerts/");
    driver.findElement(By.xpath("//a[contains(text(),'See an example')]")).click();
    Alert alert = driver.switchTo().alert();
    System.out.println(alert.getText());
    alert.accept();
    driver.quit();
  }
}

Handling Dropdowns

import org.openqa.selenium.By;
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.WebElement;
import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.support.ui.Select;

public class DropdownHandling {
  public static void main(String[] args) {
    System.setProperty("webdriver.chrome.driver", "path/to/chromedriver");
    WebDriver driver = new ChromeDriver();
    driver.get("https://www.bstackdemo.com/");
    WebElement dropdownElement = driver.findElement(By.xpath("//select"));
    Select dropdown = new Select(dropdownElement);
    dropdown.selectByVisibleText("Lowest to highest");
    driver.quit();
  }
}

Running Tests with TestNG

TestNG supports annotations, parameterized tests, and parallel execution. Add this dependency to pom.xml:

<dependency>
<groupId>org.testng</groupId>
<artifactId>testng</artifactId>
<version>7.4.0</version>
<scope>test</scope>
</dependency>

Example: TestNG Test Case

import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;
import org.testng.annotations.AfterClass;
import org.testng.annotations.BeforeClass;
import org.testng.annotations.Test;

public class TestNGExample {
  WebDriver driver;

  @BeforeClass
  public void setup() {
    System.setProperty("webdriver.chrome.driver", "path/to/chromedriver");
    driver = new ChromeDriver();
  }

  @Test
  public void openGoogle() {
    driver.get("https://www.google.com");
    System.out.println("Page Title: " + driver.getTitle());
  }

  @AfterClass
  public void quitBrowser() {
    driver.quit();
  }
}

Conclusion

Selenium combined with Java offers a robust framework for automating web testing. It handles everything from simple browser interactions to parallel test executions using Grid. With tools like TestNG, Maven, and Cucumber, you can build scalable and efficient automation solutions.
